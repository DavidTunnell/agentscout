# v0.5.8 dogfood checklist

**Goal:** prove the Option B wiring — paste API key, complete OAuth, run
analysis, receive email, click link, verify state — actually works on
real hardware against real Anthropic + real Gmail.

This checklist is the **hard gate**. CI green is necessary but not
sufficient. Until every box below is checked on a real Windows install,
v0.5.8 is not shippable to anyone external.

Date completed: __________
Tester: __________

## Pre-flight

- [ ] **Uninstall** any prior AgentScout (Add/Remove Programs).
- [ ] **Download** `AgentScout_0.5.8_x64_en-US.msi` from the
      [v0.5.8 GitHub Release](https://github.com/DavidTunnell/agentscout/releases/tag/v0.5.8).
- [ ] **Install** (SmartScreen warns; "More info" → "Run anyway"). 
- [ ] **Tray icon appears** in the system tray after install.
- [ ] **Main window opens** — Status / Recommendations / Inspector / Settings tabs all visible.

## Step 1 — API key (v0.5.5 path)

- [ ] Open Settings tab. Find "Anthropic API key" section.
- [ ] Paste your `sk-ant-...` key, click **Save** → status shows "Saved to keychain."
- [ ] Click **Test connection** → status shows "Connected — claude-sonnet-X-X replied (N input + N output tokens)."
- [ ] Close Settings, open Status tab.
- [ ] **Setup status badge** shows green for Anthropic key, yellow for Gmail (expected — not connected yet).

## Step 2 — On-demand cycle (still v0.5.5)

- [ ] AgentScout has been capturing for ≥30 minutes. Verify on Inspector tab — recent screenshots render decrypted.
- [ ] On Status tab, set "Captures from last" = 1 hour. Click **Run analysis now**.
- [ ] Result toast appears within 30-90s with "X visible recommendations" line.
- [ ] Open Recommendations tab. **At least one ranked recommendation renders** with name, tier label, evidence, build complexity, confidence %.
- [ ] Click "Implemented" / "Not Interested" / "Maybe Later" on one recommendation. Disposition updates in the row.

## Step 3 — Setup conversation (v0.5.6)

- [ ] Settings → "Personalization" section.
- [ ] Pick a starter template (e.g. "Solo engineer"). Click **Start conversation** under "Setup conversation."
- [ ] Bot opens with the skeleton + first question. Reply 3-5 times — answers should be specific to your real role.
- [ ] Click **Finalize & write profile**. Bot confirms `user-profile.md` written.
- [ ] Open `%APPDATA%\AgentScout\data\user-profile.md` in Notepad. **Confirm it sounds like you**, not generic placeholder text.
- [ ] Repeat for **Tier calibration**. `tier-definitions.json` exists and is valid JSON.
- [ ] Re-run analysis (Step 2). **Recommendations are noticeably more personalized** than the first run.

## Step 4 — Gmail OAuth (v0.5.7)

Prerequisite: a Google Cloud project with a Desktop app OAuth client + the `gmail.send` scope. If you don't have one, follow these steps in [Google Cloud Console](https://console.cloud.google.com):

1. New project (or pick existing).
2. APIs & Services → OAuth consent screen → External → fill in app name, support email, test user (yourself).
3. APIs & Services → Library → enable "Gmail API".
4. APIs & Services → Credentials → Create Credentials → OAuth Client ID → Application type "Desktop app". Note the client_id and client_secret.

Then in AgentScout:

- [ ] Settings → "Gmail" section. Note the **redirect URI** displayed (e.g. `http://127.0.0.1:51234/oauth/callback`).
- [ ] Add this exact redirect URI to your GCP OAuth client's "Authorized redirect URIs" list. Save.
- [ ] Paste **client_id** and **client_secret** in AgentScout. Click **Save creds** → green confirmation.
- [ ] Click **Connect Gmail**. Browser opens Google consent screen.
- [ ] Click "Continue" through the unverified-app warning (this is your own GCP project).
- [ ] Approve the `gmail.send` scope.
- [ ] Browser shows "Gmail connected" page. AgentScout's Settings updates within ~5s.
- [ ] Set a **recipient email** (your own Gmail). Click **Save recipient**.
- [ ] Click **Send test email**. Within 30s, an email arrives in your inbox: subject "AgentScout test email", body "If you're reading this, AgentScout's Gmail OAuth setup is working."
- [ ] **Hard pass**: email actually lands in inbox (not spam). If spam: mark "Not spam" and proceed.

## Step 5 — Email + disposition links (still v0.5.7)

- [ ] Run analysis again (Step 2). This time the result toast says "(email sent)".
- [ ] Email arrives in inbox within 30-60s. Subject like "AgentScout — N AI agent opportunities for you".
- [ ] **Open the email.** Recommendations are formatted with name, tier, evidence, action buttons.
- [ ] **Click "Show me how"** (or whatever the primary CTA is) on one recommendation.
- [ ] Browser opens to a localhost page on `127.0.0.1:<port>/disposition?...` showing "Implemented — nice." (or whatever action you clicked).
- [ ] Open AgentScout → Recommendations tab → that recommendation now has the disposition you clicked.
- [ ] **Hard pass**: link click → state change persists in DB.

## Step 6 — Auto-trigger (v0.5.8)

This step takes ~1 hour of real work to validate. Drop the threshold for fast iteration:

- [ ] Settings → Analysis section → set "Active hours threshold" = **1**. Save.
- [ ] Status tab → confirm health table shows everything green/passing.
- [ ] Use your computer normally for 1 active hour (real work — coding, browsing, writing, etc.).
- [ ] **Don't click anything in AgentScout for that hour.**
- [ ] At the 1-hour mark, AgentScout's auto-cycle loop fires `run_cycle()` automatically.
- [ ] Within ~2 minutes after the threshold, an email arrives in your inbox unprompted.
- [ ] Open the email, click a disposition link, verify recorded.
- [ ] **Hard pass**: zero manual trigger. End-to-end automatic.
- [ ] Reset threshold to **24** for production use (Settings → Save).

## Final state

- [ ] All 6 steps completed without manual code edits.
- [ ] No crashes. No silent failures. No "stub for now" text anywhere.
- [ ] System health table shows all green or expected-yellow.
- [ ] Disk state at `%APPDATA%\AgentScout\data\` contains:
      - `config.json`
      - `database.sqlite` (with non-zero `recommendations` and `cycles` rows)

## If a step fails

Capture the failure mode (screenshot + the failing step number) and report. Don't proceed past a failed step — the next step almost certainly depends on the prior one.

Common failure modes and what they tell us:
- **Step 1 fail (test connection 401)** → API key is rejected by Anthropic. Verify the key in console.anthropic.com.
- **Step 1 fail (test connection 404 model)** → Configured model name is wrong. Open Settings → Analysis → switch to a model the API actually serves.
- **Step 2 fail (Run analysis returns 0 recs)** → Captures from last hour might genuinely have no clusterable signal. Try "last 4 hours" or capture more first.
- **Step 4 fail (OAuth consent rejected)** → Redirect URI mismatch. Double-check the URI shown in AgentScout matches the GCP client's authorized redirects exactly (port included).
- **Step 4 fail (Send test email errors)** → Refresh token might not have offline scope. Disconnect, click Connect again — Google sometimes needs the second pass to issue a refresh token.
- **Step 5 fail (email lands in spam)** → Gmail-API-sent emails to yourself sometimes get filtered. Mark "Not spam"; this is a Gmail UI quirk, not an AgentScout bug.
- **Step 6 fail (auto-cycle never fires)** → Check `agentscout.log` for "auto-cycle" entries. Either the threshold isn't met (active_seconds shouldn't include idle time) or `has_anthropic_key` returned false (keychain issue).
